# 支持的后端 (Backends)

Chomsky 的后端系统负责将“已提取最优、已具体化”的 iKun(UIR) 转换为针对特定平台或硬件的产物（机器码/字节码/解释器/RTL/SQL…）。

## 已支持后端

- **[x86_64](./x86_64)**: 针对现代通用 CPU 的原生机器码生成。
- **[RISC-V](./riscv)**: 针对开放标准指令集架构 RISC-V 的机器码生成。
- **[CUDA](./cuda)**: 利用 GPU 的并行计算能力，针对计算密集型任务进行优化。
- **[SPIR-V](./spirv)**: 针对 GPU 和加速器的通用中间语言生成，支持 Vulkan/OpenCL。
- **[C](./c)**: 生成可移植的 C 代码，作为通用降级方案。
- **[JavaScript](./javascript)**: 支持 Web 平台，生成高性能的 JS。
- **[WebAssembly](./wasm)**: 针对 Web 和服务器端运行时的 Wasm 二进制格式生成。
- **[JVM](./jvm)**: 针对 Java 虚拟机生态系统的字节码生成。
- **[CLR (.NET)](./clr)**: 针对 .NET Runtime 的通用中间语言 (CIL) 生成。
- **[eBPF](./ebpf)**: 针对 Linux 内核安全执行环境的字节码生成。
- **[Verilog](./verilog)**: 针对硬件描述语言的 RTL 生成，支持高层次综合 (HLS)。
- **[SQL](./sql)**: 将逻辑转换为 SQL 查询，支持在数据库引擎中直接执行。

## 跨平台代码生成

得益于 iKun 的“意图 + 约束”表达，Chomsky 能够根据不同目标后端装配不同的 **成本模型 (Cost Model)** 与具体化策略：

- CUDA：关注线程束占用、合并访存、共享内存与同步代价
- x86_64：关注吞吐/延迟、端口压力、缓存局部性与向量化收益
- Wasm/JS：关注码大小、边界检查、运行时对象模型与 JIT 友好性
- RTL/Verilog：关注面积/时钟/时序收敛与存储带宽
